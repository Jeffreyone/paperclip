# 国际化 (i18n) 贡献指南

本文档说明如何为 Paperclip 添加新翻译和维护现有翻译。

## 目录结构

```
ui/src/i18n/
├── index.ts              # i18n 初始化和导出
├── localization-regressions.test.ts  # 回归测试
└── locales/
    ├── zh-CN.json        # 简体中文翻译
    └── en.json           # 英语翻译
```

## 添加新翻译

### 方式一：修改 JSON 文件

直接编辑 `ui/src/i18n/locales/zh-CN.json` 或 `ui/src/i18n/locales/en.json`：

```json
{
  "new.key": "新翻译内容"
}
```

### 方式二：在代码中使用翻译

1. 导入翻译函数：

```tsx
import { useTranslation } from "react-i18next";

function MyComponent() {
  const { t } = useTranslation();
  return <h1>{t("welcome")}</h1>;
}
```

2. 在翻译文件中添加对应 key：

```json
// zh-CN.json
{
  "welcome": "欢迎使用 Paperclip"
}

// en.json
{
  "welcome": "Welcome to Paperclip"
}
```

## 翻译 key 命名规范

- 使用点号分隔的层级结构，如 `page.section.component.element`
- 保持中英文 key 一致
- 使用有意义的描述性名称

### 推荐的 key 命名模式

```json
{
  "nav": {
    "home": "首页",
    "projects": "项目",
    "settings": "设置"
  },
  "button": {
    "save": "保存",
    "cancel": "取消",
    "delete": "删除"
  },
  "message": {
    "success": "操作成功",
    "error": "操作失败"
  }
}
```

## 语言切换

用户可以通过 UI 选择语言。语言设置存储在 `localStorage` 的 `paperclip-language` 键中。

### 获取当前语言

```tsx
import { getCurrentLanguage } from "../i18n";

const currentLang = getCurrentLanguage();
```

### 切换语言

```tsx
import { changeLanguage } from "../i18n";

changeLanguage("en");  // 切换到英语
changeLanguage("zh-CN");  // 切换到中文
```

## i18n 检测工具

项目集成了自动化的硬编码检测工具，用于确保没有遗漏的未翻译文本。

### 运行检测

```bash
# 运行完整分析（包括 i18n）
pnpm paperclipai analyze .

# 仅运行 i18n 检测（更快）
pnpm analyze:i18n
```

### 工具特性

- 检测硬编码的英文字符串
- 排除技术术语、代码标识符
- 排除 Tailwind CSS 类名
- 排除日志输出

### CI 集成

i18n 检测已集成到 PR 验证流程中（`.github/workflows/pr-verify.yml`），每次 PR 都会自动运行检测。

## 添加新语言

1. 在 `ui/src/i18n/locales/` 创建新的翻译文件（如 `fr.json`）
2. 在 `ui/src/i18n/index.ts` 注册新语言
3. 在 `@paperclipai/shared` 中添加语言代码定义

## 测试翻译

运行回归测试确保翻译完整性：

```bash
pnpm test:run -- ui/src/i18n/localization-regressions.test.ts
```

## 注意事项

- 所有用户可见的文本都必须使用翻译函数
- 不要在代码中硬编码任何语言特定的字符串
- 保持翻译 key 在中英文文件中同步
- 翻译内容要考虑不同语言的表达习惯
