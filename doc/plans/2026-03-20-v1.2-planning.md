### 2.2 技术债务（需关注）

1. **测试覆盖率不均**：dashboard service 有单元测试，但核心业务逻辑（checkout、budget enforcement、approval flow）缺乏充分单元测试。
2. **API 错误处理一致性**：部分路由的错误响应格式不统一（缺少统一的 error envelope）。
3. **Secret management**：schema 已定义，但 UI 端到端管理流程未完善。
4. **Document 版本**：schema 已就绪（document_revisions），但 revision diff UI 未实现。
5. **Instance settings vs Company settings**：某些配置项混淆（instance-level vs company-level）。
6. **插件生态尚未开放**：Plugin 系统核心在，但官方插件市场/文档缺失，第三方开发者无法上手。

## 3. V1.2 优先级分析

### 3.1 候选方向打分

| 方向 | 用户价值 | 工程复杂度 | 竞争差异 | 先决依赖 | 综合 |
|------|---------|-----------|---------|---------|------|
| Plugin 系统完善 + 开放 | ★★★★ | ★★★ | ★★★★★ | 无 | **高** |
| 云端 Agent（e2b/Cursor）| ★★★★ | ★★★ | ★★★★ | 无 | **高** |
| ClipMart（公司模板市场）| ★★★★★ | ★★★★★ | ★★★★★ | Plugin + export/import | 中 |
| Easy Agent Config | ★★★ | ★★ | ★★ | 无 | **中** |
| OpenClaw Onboarding | ★★★ | ★★ | ★★★ | OpenClaw adapter | 中 |
| Harness Engineering | ★★★ | ★★★ | ★★ | eval system | 低 |
| Better Docs | ★★ | ★★ | ★ | 无 | 低 |

### 3.2 V1.2 推荐主题：**Agent 生态扩展 + Plugin 开放**

理由：
1. **Plugin 系统已完成 80%，收尾成本低、价值高**。开放插件生态是 ClipMart 的前提。
2. **云端 Agent 是当前最大的用户痛点**。用户本地跑没问题，但想要 Cursor Agent / e2b 沙箱环境目前无法支持。
3. **Easy Agent Config 是上线后自然迭代**，可以随 UI 改进一起做。
4. **ClipMart 需要 Plugin + export/import 完善**，可作为 V1.2 的远期目标但 V1.3 交付。
5. **Harness Engineering** 已有 eval-harness skill 作为基础，需要集成进 Paperclip 工作流。

## 4. V1.2 目标与任务拆解

### 主题 A：Plugin 生态收尾与开放（高优先级）

**目标**：Plugin 系统从"内部功能"变为"开放能力"，第三方开发者可发布插件。

#### A1：Plugin 开发者文档（独立任务）
- 写 `doc/plugins/DEVELOPING.md`——slot 定义、lifecycle、config schema
- 写 `doc/plugins/API.md`——plugin API surface（event bridge、config、storage）
- 提供可运行的 `plugin-example` 包

#### A2：Plugin UI 管理页面完善（独立任务）
- Plugin discovery / browse 页面（即使只是静态列表也比没有强）
- Plugin 配置编辑器（schema-driven form）
- Plugin logs 查看器

#### A3：Plugin 系统加固（独立任务）
- Plugin 沙箱隔离（process isolation in production）
- Plugin 启动失败回滚
- Plugin 版本管理与 upgrade path

---

### 主题 B：云端 Agent 支持（高优先级）

**目标**：支持 e2b 和 Cursor Agent，打通"本地 process → 云端沙箱"。

#### B1：e2b Adapter（独立任务）
- 实现 `e2b` adapter type（参考 process adapter 结构）
- 支持 sandbox lifecycle（create → execute → destroy）
- 支持 credential 管理（e2b API key via secrets system）
- 文档：`doc/adapters/e2b.md`

#### B2：Cursor Agent Adapter（独立任务）
- 调研 Cursor Agent API（HTTP-based）
- 实现 cursor adapter type
- 文档：`doc/adapters/cursor.md`

#### B3：Agent Secret 管理端到端（独立任务）
- Company secrets UI（create/rotate/revoke）
- Agent config env 变量引用 secret refs
- Secret 使用审计日志

---

### 主题 C：Easy Agent Configuration（中期优先级）

**目标**：降低 agent 配置认知负担，非技术用户也能上手。

#### C1：Agent 配置向导重构（独立任务）
- 按 adapter 类型分步骤引导（而非一个大 JSON 表单）
- 提供"推荐配置"模板
- 配置预览 + 验证

#### C2：Adapter 类型说明 UI（独立任务）
- 每个 adapter 类型加 tooltip 说明
- 参数项 help text
- 常见配置示例

---

### 主题 D：OpenClaw Onboarding（中期优先级）

**目标**：让 OpenClaw 用户能一键接入 Paperclip。

#### D1：OpenClaw Adapter（独立任务）
- 参考 Claude Code adapter 模式
- 支持 OpenClaw 的 `AGENTS.md` 导入
- Onboarding wizard 中加入 OpenClaw 选项

---

### 主题 E：技术债务清理（贯穿 V1.2）

**目标**：在 feature 开发之间穿插 debt 清理，防止复杂度持续膨胀。

#### E1：核心逻辑单元测试补全（独立任务）
- checkout service 边界测试（race condition、状态机）
- budget enforcement 边界测试（并发扣款、超额场景）
- approval flow 边界测试

#### E2：API 错误响应标准化（独立任务）
- 统一 error envelope 格式 `{ code, message, details }`
- 全局 error middleware 规范化

## 8. 风险点

| 风险 | 影响 | 缓解 |
|------|------|------|
| e2b API 变动 | adapter 可能需要维护 | 用 adapter interface 隔离，API 变更只需更新 adapter |
| Cursor Agent API 不稳定 | 调研后可能发现不可行 | 先做技术调研任务，再决定是否继续 |
| Plugin 开放后收到大量低质量插件 | 生态质量受损 | V1.2 先做文档引导，暂不开放 marketplace |
| 技术债清理优先级被 feature 挤压 | debt 持续累积 | 每个 Phase 留固定比例时间给 E1 |

## 9. 不纳入 V1.2 的原因

| 方向 | 原因 |
|------|------|
| ClipMart | 依赖 Plugin 开放 + export/import 完善，建议 V1.3 |
| Harness Engineering 深度集成 | eval-harness skill 已存在，深度集成需要更多调研，V1.2 后半评估 |
| Realtime transport (SSE/WebSockets) | 复杂度高，收益不明确，暂缓 |
| Better docs（全局） | 随 feature 做，随做随写；不做专门的 docs sprint |
