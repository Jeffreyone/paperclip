# v1.2.0

> Released: 2026-03-20

## Highlights

- **Plugin ecosystem: version lifecycle and config revisions** — Two new tables (`plugin_versions`, `plugin_config_revisions`) track plugin version lineage and immutable config snapshots, enabling version upgrades and rollback of plugin configurations.
- **e2b sandbox adapter** — Cloud sandbox execution via e2b. Configure template images and startup commands for isolated agent execution. Full lifecycle support: create → execute → destroy.
- **Cursor HTTP adapter** — HTTP-based Cursor agent adapter with async polling and webhook callback support. Configure endpoint URL, auth tokens, and completion polling intervals.
- **OpenClaw adapter** — Full OpenClaw adapter with AGENTS.md import support. JSON stdout parsing for rich run transcripts.
- **Agent configuration wizard** — Refactored from a monolithic JSON form to a guided step-by-step wizard with templates per adapter type, adapter-specific runtime fields, and live config preview.
- **Standardized API error responses** — All server errors now use a consistent `{ code, message, details }` envelope via the `ApiError` class and shared `API_ERROR_CODES`.

## Improvements

- **I18n coverage expanded** — New adapter types (openclaw, cursor_http, e2b), wizard steps, plugin manager, and error messages all have zh-CN and en translations.
- **Agent `urlKey` auto-update** — Renaming an agent now automatically updates its `urlKey`. Existing urlKeys are preserved when name is not changed.
- **Plugin UI improvements** — PluginManager and PluginSettings pages refactored with schema-driven config editors and improved discovery display.
- **Test coverage expanded** — New test suites for approval flow, budget enforcement, cursor-http adapter, and issues checkout. Coverage now includes race conditions, concurrent budget scenarios, and idempotency.

## Fixes

- **Browser build compatibility** — `adapter-utils/server-utils` (Node.js-only) was incorrectly imported by `openclaw/ui/parse-stdout`. Extracted pure-JSON utilities into `adapter-utils/json-utils` to prevent Node.js modules from entering the browser bundle.

## Chores

- Database migrations for plugin versions, plugin config revisions, cursor-http, and openclaw adapters.
- Workspace dependency updates for new adapter packages across server, CLI, and UI.
- New adapter packages scaffolded under `packages/adapters/`: `cursor-http`, `openclaw`.

## Upgrade Guide

Six new database migrations run automatically on startup:

- **Migration 0031** — openclaw adapter schema additions.
- **Migration 0032** — cursor-http adapter config schema.
- **Migration 0033** — `plugin_versions` and `plugin_config_revisions` tables.

All migrations are additive — no existing data is modified. Standard `paperclipai` startup will apply them automatically.
